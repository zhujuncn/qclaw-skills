# Wizz Air Multipass 航班查询技能 v5.0

## 功能描述

使用 Wizz Air Multipass 批量查询航班可用性：
1. **自动获取航线** - 从下拉菜单提取出发城市的所有真实可用目的地
2. **分批防检测查询** - 可配置批次大小，批次间自动暂停
3. **智能延迟** - 动态延迟（2-5秒），随查询次数递增
4. **断点续查** - 自动保存进度，异常中断后可恢复
5. **智能暂停** - 连续N次异常自动暂停等待
6. **弹窗处理** - 自动关闭 WalletModal 等弹窗
7. **Excel 报告** - 生成包含有票/无票/异常的完整报告

## 使用场景

- 用户想要查询 Wizz Air 从某个城市出发的航班
- 用户想要批量查询多个目的地的可用票务
- 用户想要生成航班查询报告（Excel）
- 查找 Multipass 可用的免费航班

### 输入参数

1. **出发城市代码** - 如 `OTP`, `WAW`, `TLL`, `BEG`, `GHV`

4. **批次大小** - 每批查询目的地数量（默认 15）
5. **确认** - 输入 `y` 开始查询

## 功能特点

### 1. 分批查询机制（v5.0 新增）
- 可配置每批目的地数量（默认 15）
- 批次间自动暂停 3 分钟
- 避免触发网站安全检查
- 适合大量航线查询（60+）

### 2. 动态延迟策略（v5.0 新增）
- 基础延迟：2-5 秒随机
- 递增延迟：每 10 次查询 +0.1 秒
- 模拟人类操作节奏

### 3. 断点续查（v5.0 新增）
- 每批次结束自动保存进度到 `progress.json`
- 异常中断后可恢复
- 自动跳过已完成的目的地

### 4. 智能暂停（v5.0 新增）
- 连续 5 次异常自动暂停 60 秒
- 安全检查检测后等待 60 秒
- 避免被网站封禁

### 5. 自动获取真实航线
- 选择出发地后，自动点击 To 下拉菜单
- 提取所有可用目的地列表
- 无需手动配置目的地

### 6. 弹窗自动处理（v5.0 新增）
- 自动检测并关闭 WalletModal 弹窗
- 处理 CvoModal 等各类弹窗
- 确保表单交互不被阻挡

### 7. 用户代理轮换（v5.0 新增）
- 随机选择 Chrome/Safari/Edge
- 模拟不同设备和浏览器

### 8. Warmup 机制（v4.0 新增）
- 解决 Vue.js 表单初始化导致的按钮禁用
- 输入额外字符触发 Vue dirty 状态
- 第一个目的地不再丢失查询

## 状态分类

| 状态 | 含义 | 说明 |
|------|------|------|
| ✓ 有票 | 确认有可用航班 | 页面显示 "Your first All You Can Fly flight is covered by your Voucher" |
| ✗ 无票 | 无可用航班 | 页面显示 "Sorry, no results were found" |
| ❌ 未知页面 | 查询异常 | 安全检查/网络超时/Session过期 |
| ❌ 未找到 | 航线不存在 | 目的地不在出发地的可用航线中 |
| ❌ 按钮禁用 | 表单问题 | Vue.js dirty 状态未触发 |

## 执行流程

```
1. 用户输入出发城市代码和查询天数
     ↓
2. 检查是否有保存的进度（断点续查）
     ↓
3. 随机选择用户代理，启动浏览器
     ↓
4. 登录 Wizz Air Multipass
     ↓
5. 导航到订阅页面
     ↓
6. 自动关闭 WalletModal 弹窗
     ↓
7. 选择出发地（一次）
     ↓
8. 自动获取可用目的地列表（从下拉菜单）
     ↓
9. 重新确认出发地
     ↓
10. 分批查询主循环：
    对每批目的地：
      - 对每个目的地：
        - 选择目的地
        - 循环查询所有日期
        - 提取航班详情
        - 检测异常/安全检查
      - 保存批次进度
      - 批次间暂停 3 分钟
     ↓
11. 生成 Excel 报告（3 个 Sheet）
     ↓
12. 自动打开 Excel 文件
```

## 配置参数（v5.0）

可在脚本顶部修改以下参数：

```python
BATCH_SIZE = 15          # 每批查询目的地数量
BATCH_PAUSE_MIN = 180    # 批次间暂停时间（秒）= 3分钟
MIN_DELAY = 2.0          # 最小延迟（秒）
MAX_DELAY = 5.0          # 最大延迟（秒）
DELAY_INCREMENT = 0.1    # 延迟递增值（每10次查询）
CONSECUTIVE_ERROR_LIMIT = 5  # 连续异常阈值
ERROR_PAUSE_TIME = 60    # 异常后暂停时间（秒）
HEADLESS_MODE = False    # True = 无头模式（不显示浏览器）
```

## 注意事项

- 需要有效的 Wizz Air 账户（已配置在脚本中）
- 第一个目的地的第一次查询可能因 warmup 失败，但 warmup 解决了后续查询
- 大量查询（60+航线）可能触发安全检查，脚本会自动处理
- 安全检查期间请勿操作浏览器，等待 60 秒自动继续
- 结果仅包含 Multipass 可用航班（非普通票）
- 断点续查仅支持同一出发城市
